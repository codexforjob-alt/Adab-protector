# Adab Telegram Bot

Локальный Telegram-бот для группы, который мягко напоминает об адабе общения. Проверки сделаны на словарях, regex, капсе, флуде и повторах.

Бот следит только за формой общения:

- мат;
- прямые личные оскорбления;
- унижение собеседника;
- угрозы;
- агрессивные провокации;
- грубые насмешки;
- капс-агрессия;
- флуд;
- повторяющиеся сообщения.

Бот не является религиозным арбитром, не решает религиозные споры, ничего не удаляет, никого не банит и не мутит.

## Создание бота через BotFather

1. Откройте Telegram и найдите `@BotFather`.
2. Отправьте команду `/newbot`.
3. Укажите имя и username бота.
4. BotFather выдаст токен. Храните его только в локальном `.env`.

## Отключение Privacy Mode

Чтобы бот видел обычные сообщения в группе:

1. В `@BotFather` отправьте `/mybots`.
2. Выберите своего бота.
3. Откройте `Bot Settings`.
4. Откройте `Group Privacy`.
5. Выберите `Turn off`.

После этого добавьте бота в группу заново или проверьте, что новые сообщения стали видны боту.

## Как добавить бота в группу

1. Напишите боту `/start` в личном чате.
2. Нажмите кнопку `Добавить бота в чат`.
3. Выберите группу, в которую нужно добавить бота.
4. Отключите `Privacy Mode` в `@BotFather`, чтобы бот видел обычные сообщения в группе.
5. Запустите бота на сервере или компьютере.

## Добавление бота в группу

1. Откройте нужную группу.
2. Добавьте созданного бота как участника.
3. Админ-права не обязательны для мягких ответов, потому что бот не удаляет, не банит и не мутит.
4. Для команды `/adab_status` пользователь, который вызывает команду, должен быть администратором группы.

## Настройка `.env`

Перейдите в каталог `adab_bot`. Бот читает `.env` из этого каталога.

Скопируйте пример:

```bash
cp .env.example .env
```

На Windows PowerShell:

```powershell
Copy-Item .env.example .env
```

Заполните `.env`:

```env
BOT_TOKEN=your_telegram_bot_token
DATABASE_PATH=adab_bot.sqlite3
USER_COOLDOWN_SECONDS=120
CHAT_COOLDOWN_SECONDS=30
FLOOD_MESSAGES_LIMIT=5
FLOOD_TIME_WINDOW_SECONDS=20
CAPS_MIN_LENGTH=15
CAPS_RATIO=0.7
WEBHOOK_BASE_URL=
WEBHOOK_PATH=/telegram-webhook
WEBHOOK_SECRET_TOKEN=
PORT=8080
```

## Установка зависимостей

```bash
python -m venv .venv
```

Для Windows:

```powershell
.venv\Scripts\activate
```

Для Linux/Mac:

```bash
source .venv/bin/activate
```

Установите зависимости:

```bash
pip install -r requirements.txt
```

## Запуск

Из каталога `adab_bot`:

```bash
python main.py
```

Если `WEBHOOK_BASE_URL` не указан, бот запускается локально через polling.

Из корня проекта `PriceFinder Bot`:

```bash
python -m adab_bot.main
```

Команда `python -m pricefinder_bot.app.main` запускает старого PriceFinder-бота, а не этого бота адаба.

## Запуск на Render

На Render бот запускается как `Web Service` через webhook. Render сам задаёт `RENDER_EXTERNAL_URL`, и бот использует его для Telegram webhook.

1. Загрузите папку `adab_bot` в GitHub-репозиторий.
2. На Render нажмите `New +` -> `Blueprint` и выберите репозиторий с `render.yaml`.
3. Если создаёте вручную: выберите `New +` -> `Web Service`.
4. Укажите:
   - `Runtime`: Python;
   - `Build Command`: `pip install -r requirements.txt`;
   - `Start Command`: `python main.py`.
5. В `Environment` добавьте переменные:
   - `BOT_TOKEN`;
   - `DATABASE_PATH=/tmp/adab_bot.sqlite3`;
   - `WEBHOOK_PATH=/telegram-webhook`;
   - `WEBHOOK_SECRET_TOKEN` - любая длинная случайная строка без пробелов.
6. Нажмите `Deploy`.
7. После успешного запуска откройте URL сервиса Render. На `/health` должно быть `Adab bot is running`.

Не добавляйте `.env` и локальные базы SQLite в GitHub. Для Render используйте только переменные окружения в Dashboard.

## Команды

- `/adab_help` - кратко объясняет, как работает бот.
- `/adab_status` - показывает статус и общее число предупреждений в чате. Доступно только администраторам группы.

## Как редактировать ответы бота

1. Откройте `replies.py`.
2. Найдите нужную категорию в словаре `REPLIES`.
3. Добавьте или измените фразы.
4. Сохраните файл.
5. Перезапустите бота.
6. Для Render сделайте commit и push, чтобы сервис redeploy-нулся.

## Как настроить ответы на конкретные слова

1. Откройте `rules.py`.
2. Найдите словарь `WORD_SPECIFIC_REPLIES`.
3. Добавьте слово или фразу.
4. Укажите `category`, например `vulgar_language` или `personal_insult`.
5. Укажите короткий мягкий `reply`.
6. Перезапустите бота.
7. Для Render сделайте commit и push.

## Почему бот иногда пишет не "без мата", а "без неприличных слов"

Бот разделяет нарушения по типу:

- `profanity` - настоящий мат и матерные корни. Для этой категории ответы могут говорить "без мата".
- `vulgar_language` - неприличные, грязные, телесные или сексуальные слова, которые не обязательно являются матом. Для них бот говорит про чистоту речи и не пишет "без мата".
- `personal_insult` - личные оскорбления и переход на личности. Для них бот напоминает говорить без оскорблений.

## Редактирование правил

Основные словари находятся в `rules.py`. Там редактируются списки мата, грубой обсценной лексики, оскорблений и угроз:

- `PROFANITY_PATTERNS`;
- `VULGAR_PATTERNS`;
- `INSULT_PATTERNS`;
- `RUDE_COMMAND_PATTERNS`;
- `THREAT_PATTERNS`;
- `MOCKERY_PATTERNS`;
- `PROVOCATION_PATTERNS`.

Дополнительные большие списки находятся рядом с кодом бота:

- `words.txt` - простой список слов, каждое слово с новой строки;
- `adab_bad_words_ru.txt` - секционный список для мата, грубых слов, личных оскорблений и угроз.

Эти слова и паттерны проверяются локально.

Если нарушение не очевидное, бот молчит. Это сделано специально, чтобы не вмешиваться в обычные религиозные обсуждения и не реагировать на термины сами по себе.
